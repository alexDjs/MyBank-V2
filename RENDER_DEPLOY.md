# 🚀 MyBank V2 - Deploy to Render.com

## 📋 Pre-Deploy Checklist

### 1. GitHub Repository
```bash
# Your repository is ready at:
# https://github.com/alexDjs/MyBank-V2.git

# If you need to push updates:
git add .
git commit -m "Updated MyBank V2 for Render deploy"
git push origin main
```

### 2. Environment Variables (automatically configured via render.yaml)
- ✅ `NODE_ENV=production`
- ✅ `SECRET_KEY` (auto-generated by Render)
- ✅ `DATABASE_URL` (from PostgreSQL database)

## 🎯 Render.com Setup

### Step 1: Create New Web Service
1. Go to [render.com](https://render.com) 
2. Click **"New +"** → **"Web Service"**
3. Connect your GitHub repository: `https://github.com/alexDjs/MyBank-V2.git`
4. Select `MyBank-V2` repository

### Step 2: Configure Service (render.yaml will auto-configure)
- **Build Command:** `npm run build`
- **Start Command:** `npm start` 
- **Health Check Path:** `/`
- **Plan:** Starter (Free)

### Step 3: Database Setup (render.yaml will auto-create)
- **Database:** PostgreSQL
- **Name:** `mybank-v2-postgres`
- **Plan:** Starter (Free)  
- **Auto-linked** to web service

## 🔧 What Happens on Deploy

### Build Process:
1. **Install Dependencies:** `npm install`
2. **Initialize Database:** `npm run init-db`
   - Creates tables (users, accounts, transactions)
   - Sets up indexes and relationships
3. **Start Server:** `npm start`

### Database Schema:
- ✅ **users** table with authentication
- ✅ **accounts** table with balance tracking  
- ✅ **transactions** table with full history
- ✅ **Relationships** and constraints
